# Security Policy

## Reporting a Vulnerability

If you believe you have found a security vulnerability in this repository,
please report it privately so we can address it before public disclosure.

- **Preferred channel:** open a private security advisory via GitHub at
  https://github.com/mente/mente-worker-sandbox/security/advisories/new
- **Email:** security@mente.dev

Please include:

- A clear description of the issue and its impact.
- Steps to reproduce, including any proof-of-concept code or configuration.
- The affected version, commit, or branch.
- Your name and contact details if you would like credit.

Do **not** open a public GitHub issue, pull request, or discussion for
suspected vulnerabilities.

## Response Expectations

We aim to respond to new reports on the following timeline:

| Stage                         | Target                |
| ----------------------------- | --------------------- |
| Acknowledge receipt           | Within 3 business days |
| Initial assessment / triage   | Within 7 business days |
| Status update while ongoing   | At least every 14 days |
| Fix or mitigation available   | Within 90 days when feasible |

Timelines may vary with severity and complexity. We will keep the reporter
informed of progress and any factors that affect the schedule.

## Disclosure Policy

We follow a coordinated disclosure model:

1. The reporter privately shares the vulnerability with us using a channel above.
2. We confirm the issue, assess impact, and develop a fix or mitigation.
3. We coordinate a disclosure date with the reporter. By default, details are
   published once a fix is available, or no later than 90 days after the
   initial report, whichever comes first.
4. On disclosure we publish a security advisory that credits the reporter
   (unless they prefer to remain anonymous) and describes the issue, affected
   versions, and remediation.

Please do not publicly disclose the vulnerability, exploit it beyond what is
necessary to demonstrate the issue, or access data that does not belong to you
while we are working on a fix.

## Scope

This policy covers the code and configuration hosted in this repository.
Vulnerabilities in third-party dependencies should be reported upstream to
the responsible project; we still appreciate a heads-up so we can update our
usage.

## Safe Harbor

We will not pursue or support legal action against researchers who:

- Make a good-faith effort to comply with this policy.
- Avoid privacy violations, service disruption, and data destruction.
- Give us a reasonable opportunity to remediate before public disclosure.

Thank you for helping keep this project and its users safe.
