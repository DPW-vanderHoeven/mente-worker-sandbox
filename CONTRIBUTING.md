# Contributing

Thanks for your interest in contributing to **mente-worker-sandbox**.
This is a throwaway repo used for the MEN-118 live Mac-worker E2E run, so
contributions are usually opened automatically by the paired worker.

## Workflow

1. Create a branch off `main` for your change.
2. Keep changes small and scoped to a single concern.
3. Open a pull request against `main` describing what changed and why.

> Note: PRs are opened here only — never against `mente/mente-ios`.

## Commit messages

- Use clear, present-tense summaries (e.g. "Add contribution guide").
- Keep the subject line short; add detail in the body when needed.

## Pull requests

- Make sure the branch is up to date with `main` before requesting review.
- Explain the motivation for the change in the PR description.
- Keep each PR focused so it is easy to review.

## Questions

Open an issue if anything is unclear.
